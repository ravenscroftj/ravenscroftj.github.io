CD^2CR

2020-2021

Photo by Volodymyr Hryshchenko on Unsplash

Project Summary

As part of my PHD work I spent some time looking at ways to automatically link between news articles and scientific papers that they mention. One of the difficulties can be that language use changes significantly between the two document types. A news article might say Fossil found of dinosaur killed in asteroid strike and the scientific paper might say The Chicxulub impact played a crucial role in the Cretaceous-Paleogene extinction. How do we reconcile “asteroid strike” and “The Chicxulub impact” automatically?

The CD$^2$CR project was about building a set of news articles and scientific papers with labels that point out corresponding pairs of entities (like the asteroid/Chicxulub impact example above). I built an annotation tool using ReactJS on the frontend and Flask on the backend. We have also made the dataset produced freely available to download via github too.

SAPIENTA

2013-Present

Photo by Russ Ward on Unsplash

Project Summary

The SAPIENTA Project originally led by Prof Maria Liakata was first presented in a journal paper in 2012. The system provides automatic annotation of Core Scientific Concepts in papers at sentence level.

In 2013, I took over maintaining SAPIENTA. I was employed by Prof. Liakata through the University of Warwick to rewrite the core system in Python (the original was PERL-based) and I also added some enhancements including an improved web service, automatic PDF to XML conversion and a web UI.

We still run the web service in two places: there is the primary service hosted privately by us via the Partridge Project here and there is a mirror site graciously hosted by the Aberystwyth University Computer Science Department.

The code, models and docker plumbing for running SAPIENTA can all be found over on Github

indiescrobble.club

2021-Present

Photo by Alex Litvin on Unsplash

Project Summary

Since December 2021 I’ve been very interested in the Indieweb movement and technologies. I watch a lot of TV and movies and listen to a lot of music and I wanted a way to be able to keep a diary of these things through my blog. So I built IndieScrobble - a system that allows you to log what you watched/listened to, how you rated it and give a little commentary and post this back to your website via micropub.

It was also an opportunity to have a go at building out a web app using GoLang which I had been meaning to try for a while.

You can find the code for indiescrobble on Github.
